## 2026-05-24 - [Interactive Map Layout Bug]\n**Learning:** When using absolutely positioned elements (like markers) on top of an SVG map, the parent container must enforce the same aspect ratio as the SVG's viewBox, or the SVG must be set to `preserveAspectRatio="none"`. If the container's aspect ratio changes (e.g. using fluid heights), the SVG paths and HTML markers will drift apart due to letterboxing.\n**Action:** Ensure SVG map containers use Tailwind's `aspect-[]` classes to lock the coordinate system in place, or place interactive elements inside the SVG coordinate system.
